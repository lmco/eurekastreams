## Overview

Eureka Streams is enabled to accept e-mails that will post a message to a stream or comment on a post.  Users will be able to add to their contacts list the e-mail address of individual streams.  Users will be able to reply to post notifications in order to comment on that particular post.

## Conditions of satisfaction

### Follow/Unfollow

1. The Knowledge Worker must be able to follow any public stream.
2. The Knowledge Worker must be able to stop following any stream he is following.

### Email Subscription

3. The Knowledge Worker must be able to subscribe via email to any stream
4. The Knowledge Worker must be able to unsubscribe from email to any subscribed stream.

### RSS Feed

5. The Knowledge Worker must be able to view an ATOM feed for a stream.
6. The Knowledge Worker must see for each activity the user name and activity content and for the feed to be sorted by activity date
7. The system must authenticate the ATOM feeds.

### Bookmarks 

8. The Knowledge Worker must be able to bookmark any stream
9. The Knowledge Worker must be able to reorder his bookmarks
10. The Knowledge Worker must be able to remove a bookmark.

### Add to Start Page

11. The Knowledge Worker must be able to create an app from a stream, custom stream, or group stream.
12. The Knowledge Worker must be redirected to the start page after creating an app from a stream, custom stream, or group stream.

### Filters

13. The Knowledge Worker must be able to create a filter that includes one or more colleagues, groups, or predefined streams and optional keywords.
14. The Knowledge Worker must be warned when the title for a filter is not provided.
15. The Knowledge Worker must be able to edit a filter.
16. The Knowledge Worker must be able to delete a filter.
17. The Knowledge Worker must be able to reorder filters.

### Default Streams

18. The Knowledge Worker must be able to view a stream of activity from a predefined stream representing the streams he is following.
19. The Knowledge Worker must be able to view a stream of activity from a predefined stream representing everyone in the system. 
20. The Knowledge Worker must have one undeletable filter to his liked items.
21. The Knowledge Worker must have one undeletable bookmark to his personal stream.

### Adding an Email Contact for a Stream

22. The Knowledge Worker must be able to select, for a stream, a link that will
add a contact in their contact list.

### Replying to a post notification e-mail

01. The Knowledge Worker must be able to "reply to "an e-mail received from a post in order to send an e-mail to the Eureka Streams e-mail box.


### E-mail posting or commenting

01. The Knowledge Worker must be able to post a message to a stream via e-mail. 
02. The Knowledge Worker must be able to comment on an activity in stream via e-mail. 
03. The Knowledge Worker must be able to e-mail with a signature, original e-mail text, etc. and have these extra itemsremoved allowing only the post or comment text to be processed.
04. The Knowledge Worker must be able to send an e-mail that includes a multipart form and/or HTML and have the HTML stripped out and/or the additional multipart forms ignored allowing only the post or comment text to be processed.
05. The Knowledge Worker must be able to e-mail a post larger than 250 characters and have the extra characters appear as a comment below their post.
06. The Knowledge Worker must be able to e-mail a comment longer than 1,000 characters and have the extra characters appear as additional comments below their comment.

## User experience

### Following a Stream

![](images/Subscribe_Follow.png)

1. A user is able to follow any stream (except his own) by clicking on the "Follow" button on their stream.

![](images/Subscribe_Follow_Menu.png)

1. When the user hits the "Follow" button a Subscribe modal pops up. Checking "Add this Stream to my Start Page" will add the stream to the users start page after they click "Save Changes".
2. Checking "Bookmark this Stream" will bookmark the stream after the user clicks "Save Changes"
3. Checking "Notify my via Email" will sign the user up to receive email alerts on posts to the stream when the user clicks "Save Changes"
4. Clicking "Save Changes" executes the changes checked.
5. Clicking "No Thanks" closes the modal and ignores all of the check boxes

![](images/Subscribe_Unfollow.png)

1. A user is able to unfollow any followed stream by clicking on the "Unfollow" button. The Unfollow button shows up in the same place the follow button does.

### Managing Stream Subscriptions

![](images/Subscribe_Menu.png)

1. Hovering over the subscriptions icon pops up the subscriptions menu.
2. Clicking "Bookmark Stream" adds the stream to the user's bookmarks
3. Clicking "Add to Start Page" adds the stream to the user's start page.
4. Clicking "Subscribe via Email" signs the user up to receive emails for every post to the stream. When clicked the menu option should change to "Unsubscribe to email"
5. Clicking "RSS Feed" opens the streams RSS feed in a new window.
6. Clicking "Save as Contact" opens the save to outlook dialog defined below.

### Managing Bookmarks

![](images/Subscribe_Add_Bookmark.png)

1. Bookmarks can be added by clicking on "Bookmark a Stream". When clicked, a search box opens up allowing the user to instantly search for any stream.
2. Hovering over the bookmark shows a "+" sign that when clicked adds the stream to the users bookmarks.

![](images/Subscribe_Remove_Bookmark.png) 

1. Hovering over bookmarks shows an "X" icon that when clicked removes the bookmark from the user's list.

### Managing Filters

![](images/Subscribe_Filter_Hover.png)

1. Hovering over a filter shows an edit and delete button. Clicking edit brings up the edit modal defined below. Clicking the delete button deletes the filter.

### Adding a Filter

![](images/R1_PS_F16_managelists_CreateNewSavedSearch.png)

1. Input box to enter a name for the custom stream.  Name fields supports up to 50 characters.
2. The Knowledge Worker can select the following options: Everyone, Following, Groups I've Joined, Posted To, Authored By, and Liked By.
3. Input box to enter one or more keywords.  Keywords field supports up to 50 characters per keyword and an infinite number of characters.
4. When a new list is successfully saved, a warning message will appear at the top of the page telling the user: "Your stream has been successfully saved" If one or both of the required fields are left blank the following message will appear above the form: "Name field is required" and "At least one stream is required"

![](images/R1_PS_F16_managelists_CreateNewSavedSearch_fig02.png)

1. The Knowledge Worker can select the empty text box to input 1 to many people or group streams by typing in their name and selecting the stream.

### Editing a Filter

![](images/R1_PS_F16_managelists_EditSavedSearch_fig01.png)

1. Input box to edit a stream name.
2. The Knowledge Worker can select a predefined or custom stream.
3. Input box to enter one or more keywords.
4. When a new stream is successfully saved, a warning message will appear at the top of the page telling the user: "Your stream has been successfully saved" If one or both of the required fields are left blank the following message will appear above the form: "Name field is required" and "At least one stream is required"
5. Clicking the delete icon displays a pop-up window asking the Knowledge Worker the following: "Apps based on this stream will need to be manually deleted from the Start Page.  Are you sure you want to delete this custom stream?"  If the Knowledge Worker clicks "yes" the stream is deleted and a warning message is displayed with a close link: "The custom stream has been deleted".

Note: There are additional scenarios where a custom stream is automatically modified or deleted.

1. If a Knowledge Worker leaves a private group and a stream is based on the private group's stream, automatically delete the stream.
2. If a Knowledge Worker leaves a public group, do not delete the stream.
3. If a Public or Private group is deleted and a stream is based on the private group's stream, automatically delete the stream.
